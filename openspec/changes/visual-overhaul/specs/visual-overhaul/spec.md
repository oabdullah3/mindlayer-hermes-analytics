# Visual Overhaul

**Purpose**: Fix visual bugs, UX gaps, and data presentation issues in the Streamlit dashboards surfaced during user testing with 117 real sessions.

## ADDED Requirements

### Requirement: KPI cards display fully without truncation

The system SHALL render all KPI metric cards in the portal home page and session detail page with sufficient width to display their full values, without truncating dates, model names, or numeric values.

#### Scenario: Updated date fully visible

- **WHEN** the portal home page renders the "Updated" metric card
- **THEN** the date value displays in full (e.g., "2026-06-24 06:53") without ellipsis truncation

#### Scenario: Session detail header metrics fully visible

- **WHEN** the session detail page renders the Model, Platform, Started, Ended, and Duration metric cards
- **THEN** all five metric values display fully without truncation, including model names up to 40 characters, date timestamps, and duration strings

### Requirement: Histogram and bar charts have spacing between bars

The system SHALL render all histogram and bar charts with a visual gap between adjacent bars to prevent bars from appearing fused together.

#### Scenario: Token estimate histogram shows bar gaps

- **WHEN** the Skills Analytics page renders the token estimate distribution histogram
- **THEN** each bar has a visible gap (bargap ≥ 0.15) separating it from its neighbors

#### Scenario: Tool calls per session histogram shows bar gaps

- **WHEN** the Tools Analytics page renders the tool call count distribution histogram
- **THEN** each bar has a visible gap (bargap ≥ 0.15) separating it from its neighbors

### Requirement: Pie chart uses hover tooltips and Golden Ratio colors

The system SHALL render the tool call distribution pie chart without a sidebar legend, using hover tooltips for detail and colors generated by the Golden Ratio algorithm for maximum visual distinction.

#### Scenario: No sidebar legend on pie chart

- **WHEN** the Tools Analytics page renders the tool call distribution pie chart
- **THEN** no sidebar legend is displayed; instead, hovering over a slice shows a tooltip with tool name, call count, and percentage share

#### Scenario: Golden Ratio colors for many slices

- **WHEN** the pie chart has more than 10 slices
- **THEN** each slice has a perceptually distinct color generated from the Golden Ratio conjugate ($\phi^{-1} \approx 0.618$) applied to HSL hue rotation

### Requirement: Chart legends do not overlap with toolbar controls

The system SHALL position chart legends so they do not visually overlap with Plotly's modebar controls (download, zoom, pan, etc.) or with chart data.

#### Scenario: Skills timeline legend below chart

- **WHEN** the Skills Analytics page renders the stacked skills usage timeline
- **THEN** the legend is positioned horizontally below the chart area, not overlapping the modebar controls at top-right

#### Scenario: Tools timeline legend below chart

- **WHEN** the Tools Analytics page renders the stacked tools usage timeline
- **THEN** the legend is positioned horizontally below the chart area, not overlapping the modebar controls

### Requirement: Session table uses single-row selection with clear affordance

The system SHALL use single-row selection mode in the session overview table, with a clear visual affordance that only one session can be selected at a time.

#### Scenario: Clicking a session row selects only that row

- **WHEN** the user clicks a row in the session overview table
- **THEN** only that row is highlighted; previously selected row is deselected

#### Scenario: Selection shows toast with navigation hint

- **WHEN** the user selects a session row
- **THEN** a toast notification appears instructing the user to navigate to the Session Detail page via the sidebar

### Requirement: Session display names include title

The system SHALL display the session title (chat_name) alongside the session ID in all session lists, dropdowns, and detail headers to provide human-readable context.

#### Scenario: Session dropdown shows title and ID

- **WHEN** the Session Detail page renders the session selector dropdown
- **THEN** each option displays the session title (or "Untitled" if NULL) followed by the truncated session ID

#### Scenario: Session detail header shows title

- **WHEN** a session is selected in Session Detail
- **THEN** the page heading includes the session title (e.g., "Organize Desktop Projects") as the primary identifier

#### Scenario: Session table row shows title

- **WHEN** the Session Overview table renders
- **THEN** each row includes a "Title" column showing the session's `chat_name`

### Requirement: Truncated messages show truncation indicator

The system SHALL append a visible "… (truncated)" indicator when user message content exceeds the display limit, and render messages in visually distinct styled containers.

#### Scenario: Long message shows truncation indicator

- **WHEN** a user message exceeds 300 characters
- **THEN** the displayed text ends with "… (truncated)" to indicate content was cut

#### Scenario: Messages rendered in border containers

- **WHEN** the Session Detail page displays user messages
- **THEN** each message is rendered in a bordered container with the timestamp as a header, visually distinct from surrounding content

### Requirement: Tool calls table shows name, count, and success rate

The system SHALL render the tool calls table in Session Detail with tool name, call count, and aggregated success rate, without displaying the raw message_ids column.

#### Scenario: Tool calls table structure

- **WHEN** a session has tool calls
- **THEN** the table displays columns: Tool Name, Call Count, and Success Rate (percentage of calls with exit code 0)

### Requirement: Drill-down tables include session title

The system SHALL include the session title in skill drill-down and tool drill-down result tables alongside the session ID and date.

#### Scenario: Skill drill-down shows session titles

- **WHEN** the user selects a skill in the Skills Analytics drill-down
- **THEN** the results table includes a "Session Title" column showing the chat_name for each session that loaded the skill

#### Scenario: Tool drill-down shows session titles

- **WHEN** the user selects a tool in the Tools Analytics drill-down
- **THEN** the results table includes a "Session Title" column showing the chat_name for each session that used the tool

### Requirement: Tools chart has Show All toggle and handles long labels

The system SHALL provide a "Show all tools" toggle for the tools bar chart (defaulting to top 10) and SHALL ensure long tool names do not overlap with chart controls.

#### Scenario: Default view shows top 10 tools

- **WHEN** the Tools Analytics page renders the tools by call count bar chart
- **THEN** only the top 10 tools are displayed, with a "Show all tools" checkbox available

#### Scenario: Labels with long tool names are legible

- **WHEN** tool names exceed 40 characters in the bar chart
- **THEN** labels are either wrapped, truncated with ellipsis, or the chart margin accommodates the full label without overlapping the modebar
